# Как получить все изменения из удаленного репозитория

Чтобы получить все изменения из удалённого репозитория, включая:
  - новые коммиты в существующих ветках
  - новые ветки, созданные другими студентами
  - обновления ссылок на удалённые ветки

используйте команду:
```git fetch --all```

Что делает ```git fetch --all```
  - загружает все изменения со всех удалённых репозиториев (обычно это origin)
  - **не изменяет ваши локальные ветки**
  - обновляет ссылки вида origin/branch-name

Например, после этого вы сможете увидеть новые ветки:

  - ```git branch -r``` - выводит список всех удалённых веток.
По умолчанию команда выводит список локальных веток.

  - или создать локальную ветку на основе удалённой:
```git checkout -b new-branch origin/new-branch```


## Разница git fetch с git pull
  - git fetch — только скачивает изменения
  - git pull — *скачивает и сразу пытается слить (merge) в текущую ветку*

**В командной работе обычно безопаснее сначала делать git fetch, чтобы посмотреть изменения.**

### Типичный рабочий процесс:

git fetch --all
git branch -r
git checkout branch-name
git pull
